Private crypt hashing for phpBB 3

public

Parameters:
stringPassword
stringSettings
stringHash-lookup
Returns:
string phpBB3 password hash

We're kind of forced to use MD5 here since it's the only cryptographic primitive available in all versions of PHP currently in use. To implement our own low-level crypto in PHP would result in much worse performance and consequently in lower iteration counts and hashes that are quicker to crack (by non-PHP code).
